# 功能总览

Visual Worktree 围绕一个核心模型设计：源项目仍保留在原来的仓库目录，需求开发发生在 `worktreesRoot/{任务名}/{项目名}` 下的 worktree 中。应用把分散的 Git 状态、任务进度、环境检查和外部链接合到一个桌面工作台里。

## 项目扫描

项目视图会扫描你配置的源项目根目录，只读取一级 Git 仓库，并自动跳过隐藏目录和忽略列表。每个项目会展示当前分支、是否为主分支、未提交文件数量、ahead/behind、远程跟踪分支、fetch 失败状态和关联 worktree。

扫描可以选择是否自动 fetch。关闭 auto fetch 时刷新更快，适合日常查看本地状态；打开后可以更准确地看到远程 behind，但会受网络、VPN 和 SSH 鉴权影响。

## 项目操作

每个项目可以直接打开 Finder、VSCode 或终端。VSCode 默认使用新窗口打开，避免替换你当前正在看的窗口；终端支持 macOS Terminal、iTerm2 和 Ghostty，并在打开失败时回退到系统 Terminal。

项目还支持隐藏和置顶。隐藏用于把不常看的仓库从日常视图里收起来，置顶用于把高频仓库放到前面。隐藏项目不会参与统计和批量操作，避免误操作。

## Git 状态判断

应用会判断当前分支是否属于主分支候选列表，默认候选是 `master` 和 `main`。切回主分支时会先检测仓库实际存在的主分支，再选择可用分支，避免仓库用 `main` 时硬切 `master` 报错。

未提交变更、未推送提交和落后远程都会以可扫描的标签展示。项目有 GitLab remote 时，应用会从 origin 地址推导 GitLab 项目页面，方便从任务行或项目行直接打开。

## 批量操作

项目视图支持批量切回主分支、批量拉取更新和批量暂存变更。批量操作逐项执行，并通过进度条展示当前处理到哪个项目。单个项目失败不会阻断后续项目，结果会按项目返回，适合一次整理多个仓库工作区。

## 任务化 worktree

Worktree 视图按任务分组展示 worktree。推荐结构是：

```text
worktreesRoot/{任务名}/{项目名}
```

同一个任务可以包含多个项目 worktree。应用会扫描所有源项目登记过的 worktree，找出位于 worktreesRoot 下的路径，并按任务目录聚合。任务名可以包含 `/`，应用会保留完整任务路径。

## 创建 worktree

创建任务时可以选择多个项目，应用会为每个项目创建对应 worktree。新建分支默认基于配置里的主分支候选，并优先使用远程主分支引用，避免源仓库当前停在其他分支时把新分支建错起点。

创建 worktree 时会跳过 Git hooks，减少 husky 等项目 hook 在新 worktree 依赖未安装时导致创建失败的概率。前端或 Node 项目如果源仓库已有 `node_modules`，应用可以在 worktree 里创建软链接，减少重复安装依赖。

## 删除和清理

单个 worktree 可以安全删除。有未提交变更时，应用会先阻止危险删除，再让你二次确认是否强制删除。任务级删除会先归档工作文档，再删除任务目录，并写入历史记录。

清理建议会扫描可安全删除的 worktree，帮助你找出已经回到主分支或不再需要的任务目录。失效 worktree 可以执行 prune 清理。

## 任务状态

任务可以手动标记状态，默认状态是「未开始」。内置状态包括开发中、自测中、待提测、测试中、待发布和已发布。任务列表可以按名称或状态排序，适合把正在推进的工作放在更醒目的位置。

## 任务链接

每个任务可以挂多个外部链接，并为链接填写展示名。常见用法包括 Jira、飞书需求、测试用例、设计稿、上线单和排查文档。链接会持久化到本地，点击后通过系统浏览器打开。

## 卡点备注

任务可以记录卡点备注，适合保存当前阻塞原因、等待谁确认、需要补什么数据或下一步要处理什么。卡点跟随任务展示，不需要回聊天记录里翻上下文。

## 需求流程

每个任务都有一组可配置流程步骤。步骤可以打勾标记完成，也可以配置执行命令。执行命令会在任务目录下运行，并支持 `{path}`、`{task}`、`{branch}` 等占位符，参数会用 shell 引号保护，避免任务名或路径里的空格、中文、`&` 等字符拆坏命令。

步骤执行会实时展示 stdout/stderr 输出。命令成功后可以自动勾选步骤；失败时可以停止后续批量步骤，并保留最近一次输出，方便回来查看。

## 运行全部流程

配置了命令的流程步骤可以一键运行全部。运行期间每个步骤有独立 loading 状态，不会阻塞其他任务的普通操作。失败输出会标记在任务流程入口上，让你快速看到最近未通过的步骤。

## 工作文档

应用支持工作文档模板。新建任务或项目 worktree 时可以自动创建指定目录或文件，例如 `docs/`。删除任务前会把工作文档归档到本地持久化目录，避免任务清理时把过程记录一起删掉。

## 环境健康检查

环境检查会按任务目录识别项目，并检查依赖、端口、服务连通性和 Git 状态。结果会汇总成任务级状态：环境正常、环境检查中或存在环境问题。检查结果会缓存到本地，重启后仍能看到上次状态。

你可以配置环境检查角色，把任务目录下的不同子目录映射为前端、后端或其他角色。没有明确角色时，应用会自动扫描任务目录下的项目。

## 看板视图

看板视图基于任务状态组织任务，把未开始、开发中、自测、测试、待发布和已发布等阶段变成更适合巡检的视图。它适合在任务较多时看整体分布，而不是逐行扫列表。

## 流程视图

流程视图把任务流程集中展示，方便横向查看每个任务做到哪一步。它适合做需求推进检查，例如哪些任务已经完成自测，哪些任务还缺发布前检查。

## AI 用量概览

应用可以读取本机 Claude Code 会话数据，并按任务汇总 token 和费用。这个能力只做本地统计，用来帮助你了解某个任务的 AI 使用成本，不会上传会话内容。

## 历史任务

任务删除后会写入历史记录，保留任务名、任务状态、需求链接和归档工作文档路径。历史任务入口可以帮助你找回已经清理掉的任务上下文。

## 设置项

设置里可以配置源项目根目录、worktree 根目录、主分支候选、忽略项目、是否自动 fetch、CI/CD 链接、编辑器命令、终端应用、流程步骤、环境检查角色、工作文档模板和任务标题徽标。

所有这些设置都存储在本地配置目录，适合按个人工作习惯调整。

## 桌面集成

Visual Worktree 是 Electron 桌面应用，渲染层负责界面，主进程负责 IPC 转发和系统能力。它可以调用系统 Git、Finder、VSCode、终端、剪贴板和外部浏览器。核心 Git 逻辑在纯 Node 模块中，便于用真实临时 Git 仓库做集成测试。

## 本地优先

应用默认把配置、任务状态、任务链接、流程进度、卡点、环境检查缓存、历史任务和流程输出保存到 `~/.visualWorktree`。仓库内容、任务路径和状态不会上传到远端服务。
